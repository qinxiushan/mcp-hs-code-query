## 📋 文档命名规范

### CHANGELOG（修改记录）
格式: `CHANGELOG_XXX_简短描述.md`

示例:
- `CHANGELOG_001_修复数据解析和URL问题.md`
- `CHANGELOG_002_添加代理支持.md`
- `CHANGELOG_003_优化查询性能.md`

### FEATURE（新功能）
格式: `FEATURE_XXX_功能名称.md`

示例:
- `FEATURE_001_添加Excel导出功能.md`
- `FEATURE_002_支持多网站查询.md`

### BUGFIX（问题修复）
格式: `BUGFIX_XXX_问题描述.md`

示例:
- `BUGFIX_001_修复网络超时问题.md`
- `BUGFIX_002_修复中文编码错误.md`

### OPTIMIZATION（性能优化）
格式: `OPTIMIZATION_XXX_优化内容.md`

示例:
- `OPTIMIZATION_001_查询速度优化.md`
- `OPTIMIZATION_002_内存使用优化.md`

## 📖 文档模板

每个文档应包含以下章节：

1. **基本信息**
   - 日期
   - 版本
   - 修改人
   - 类型

2. **问题描述**
   - 现象
   - 日志信息
   - 原因分析

3. **修复/改进方案**
   - 原有代码
   - 修改后代码
   - 技术说明

4. **影响范围**
   - 修改的文件
   - 影响的功能

5. **测试验证**
   - 测试用例
   - 预期结果

6. **后续建议**（可选）

7. **版本历史**
